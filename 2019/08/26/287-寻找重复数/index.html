287.寻找重复数

Posted on 2019-08-26 | In Algorithm

给定一个包含 n + 1 个整数的数组 nums，其数字都在 1 到 n 之间（包括 1 和 n），可知至少存在一个重复的整数。假设只有一个重复的整数，找出这个重复的数。

示例 1:

输入: [1,3,4,2,2]
输出: 2
示例 2:

输入: [3,1,3,4,2]
输出: 3
说明：

不能更改原数组（假设数组是只读的）。
只能使用额外的 O(1) 的空间。
时间复杂度小于 O(n2) 。
数组中只有一个重复的数字，但它可能不止重复出现一次。

思路：最开始的时候第一个想到的方法就是hash，但是不满足空间复杂度为O(n)。然后又想到使用排序，再根据二分搜索。但是这种时间复杂度也不小。后面才知道这是一张龟兔赛跑问题。

最近写leetcode 287时，接触到了“龟兔赛跑算法”，搜索了很多文章，终于透彻地理解，现在用最通俗易懂的方法记录一下：
判断是否有环的关键思想是：定义两个指针，指针都在开始，一个指针走得快叫fast指针，一个走得慢叫slow指针，fast和slow一直走，在某个时刻，如果fast可以遇到slow，则存在环。
上面的思想很好理解，通俗来说，假如两个人比赛跑步，在一个400米的环跑道上，跑的快的一定会再遇到跑的慢的，跑的快的同学跑的路程比慢的同学多整数多个圈，可能多400，800，1200，……。然而如果没有环，则跑得快的跑到终点也不会碰到跑的慢的。

对于速度来说，检测环是否存在，fast>slow即可，一般都用 fast = 2slow，即fast一次走两步，slow走一步.

假设fast的速度为slow的2倍，如果fast走过了整个链表，没有遇到slow，则没有环
否则，有环，假设fast和slow在环的M相遇：
则相遇时：(n表示环的长度，即理解为周长)
1>slow走过的路程为：sl = |SO| + |OM| + an,(a=1,2,3,4,….)
2>fast走过的路程为：sf =|SO| + |OM| + bn (b=1,2,3,4,..)
(其中sf=2sl,b>n,因为速度快2倍)
3>用2式减1式，sf-sl = sl = (b-a)*n,即slow或fast相遇时,两者走的总路程是环长度的整数倍。

利用第三点，我们可以求切点（环入点）
切入点是 s走m步到的o点，我们定义一个指针指向s,记为s1。
接着求出fast和slow相遇的M点，用一个指针指向M，记为s2.
两个指针同时走。当走到m步时，s1走过s1+m，即指向o。而s2所指向的是sl+m,有上面第三点可知，sl = (b-a)*n，为环的整数倍，所以sl+m也指向o点，即：相遇点走到环入点的距离=起始点走到换入点的距离，表示为|SO| = |MO|.
